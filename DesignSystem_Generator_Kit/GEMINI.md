# Gemini CLI - 設計系統生成器行為準則 (GEMINI.md)

本文件定義了此工具包在生成設計規範時必須遵守的核心原則與用語規範。

## 1. 核心工作流程
- **先分析，再實作**：在生成任何程式碼或正式規範前，必須先提供分析結果與規劃藍圖，經使用者確認後方可執行。
- **自動化優先**：盡可能產出可直接被機器讀取 (JSON) 或 AI 繪圖 (Prompt) 的內容。

## 2. 語言與用語規範 (繁體中文 - 台灣)
必須嚴格遵守台灣 UI/UX 業界與學術界的習慣用語，禁止使用中國大陸用語。

| 類別 | 推薦用語 (台灣) | 禁止使用 (大陸用語) |
| :--- | :--- | :--- |
| **研究** | 使用者研究、訪談、易用性測試 | 調研、用戶體驗測試 |
| **流程** | 全流程、端到端、工作流 | 全鏈路、閉環 |
| **技術** | 智慧、生成式 AI、演算法 | 智能、AIGC、算子 |
| **設計** | 介面、元件、屬性、樣式 | 界面、組件、特性、樣式 |
| **互動** | 點擊、選取、觸發、回饋 | 點擊、勾選、喚起、反饋 |

## 3. 學術與專業標準
- **學術規範**：所有專業術語與標題命名應符合台灣國立大學碩博士論文之嚴謹程度。
- **UI 規範**：遵循 iOS Human Interface Guidelines (HIG) 與 Android Material 3 規範。

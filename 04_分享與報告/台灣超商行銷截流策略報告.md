# **台灣超商精準行銷與截流策略報告**

**報告日期：** 2026年2月

**專案代號：** FamiPort-Opt-2026

**研究對象：** 7-ELEVEN (OPENPOINT)、萊爾富 (Hi-Life)、OK Mart (OK go)、第三方發票 App (發票怪獸/AIFIAN)

**關鍵字：** 零售媒體聯播網 (RMN)、情境式行銷 (Situational Marketing)、競品截流 (Conquesting)、OMO 閉環、遊戲化 (Gamification)

## ---

**1\. 執行摘要 (Executive Summary)**

本報告旨在協助 My FamiPort App 團隊進行產品優化，核心目標在於解決「通知騷擾（Notification Fatigue）」痛點，並透過發票數據的應用實現「競品截流（Competitor Conquesting）」。在台灣高度飽和的便利商店市場中，App 已不僅是服務工具，而是零售媒體聯播網（RMN）的核心戰場。

研究發現，台灣零售 App 市場已進入「場景爭奪戰」階段。7-ELEVEN 透過 OPENPOINT 生態系，成功將「工具型操作」（如繳費、取貨）轉化為「高價值流量入口」，其推播策略已從全網廣播轉向「行為觸發（Event-Triggered）」，顯著降低了用戶反感度並提升轉換率。相比之下，萊爾富與 OK Mart 則採取差異化策略，前者利用異業票券維持開啟率，後者則深耕 Kiosk 與 App 的功能性連結。而第三方發票 App 如「發票怪獸」則展示了如何利用遊戲化（Gamification）機制低阻力地獲取用戶消費數據，並精準地將競品客群標籤化，進行二次銷售。

本報告將分為四大章節，地毯式拆解各競品的 UI/UX 流程、數據變現邏輯及截流防禦機制，並針對 My FamiPort 提供具體優化建議。

## ---

**2\. 7-ELEVEN (OPENPOINT)：生態系精準推播與情境閉環**

7-ELEVEN 的數位策略核心在於將 OPENPOINT App 打造為「生活服務的遙控器」，而非單純的集點工具。其最大的優勢在於將線下行為（ibon、POS 結帳）與線上數據（App ID）進行了近乎即時的串聯，構建了一個強大的情境式行銷閉環。

### **2.1 生態系標籤與分眾機制 (Ecosystem Tagging)**

OPENPOINT 的精準推播基礎來自於三方數據的即時整合，這使其能夠超越基礎的人口統計學標籤，進入行為心理學的範疇。

#### **2.1.1 數據源整合架構**

OPENPOINT 能夠實施精準推播，關鍵在於其後端數據池（Data Lake）整合了三個維度的即時訊號：

* **支付數據 (Transaction Data)：** 透過 icash Pay 或 OPEN 錢包，系統不僅記錄消費金額，更捕捉商品類別（SKU Level）。例如，系統能識別出某用戶是「早晨咖啡族」或「週末家庭採購族」。  
* **服務數據 (Service Interaction)：** 這是 7-ELEVEN 最強大的護城河。透過 ibon 機台的繳費（水電、卡費、罰單）、票券購買與列印服務，App 能捕捉到用戶的「生活剛需」時刻。  
* **物流數據 (Logistics Signals)：** 透過交貨便與賣貨便系統，App 掌握了用戶的取件頻率與網購平台來源。這些數據點被用來判斷用戶的「到店預期時間」。

#### **2.1.2 情境式標籤 (Situational Tagging) 策略**

與傳統靜態標籤不同，OPENPOINT 強調「當下狀態（Contextual State）」。

* **「剛需」標籤：** 當系統偵測到用戶有一筆「待取包裹」或「即將到期帳單」時，該用戶會被暫時歸類為「高活躍預備軍」。此時的推播內容會從「促銷」轉為「服務提醒」，降低用戶的防禦心。  
* **「剛完成」標籤：** 當用戶剛在門市完成特定動作（如取貨），系統會觸發即時回饋機制，這是介入行銷的最佳時機。

### **2.2 包裹取件的情境行銷：從工具到變現**

針對「通知騷擾」痛點，OPENPOINT 的解法是\*\*「將廣告包裝成服務通知」\*\*，並利用取貨流程創造高強度的 App 開啟需求。

#### **2.2.1 取件前通知與 UI 誘因**

當包裹抵達門市時，App 會發送推播通知。與競品不同的是，OPENPOINT 的通知文案通常伴隨著具體的利益點，例如「您的包裹已送達，取件即享運費抵用券」或「取件限定：咖啡半價」。這種設計讓通知不再是單純的資訊告知，而是一個「領獎通知」。

#### **2.2.2 關鍵 UI 互動流程：0元包裹認證**

這是 OPENPOINT 最具殺傷力的 UI 設計，成功將線下取貨人流 100% 轉化為 App 活躍用戶 1。

* **場景：** 用戶領取「已付款（0元）」包裹，傳統流程需出示身分證。  
* **OPENPOINT 優化流程與 UI 描述：**  
  1. **入口 (Entry Point)：** 用戶點擊 App 首頁的「寄取包裹」圖示，或直接點擊取件推播通知。  
  2. **查詢 (Query)：** 進入「我的包裹」頁面，點選「取件查詢」。  
  3. **匯入 (Import)：** 用戶點選「匯入取件包裹」，系統自動驗證手機號碼與會員帳號的關聯性。  
  4. **條碼生成 (Barcode Generation)：** 系統生成一個動態的「0元認證條碼」。  
  5. **行銷置入 (Native Ad Placement)：** **這一步是關鍵。** 在展示認證條碼的畫面中，螢幕下方約 1/3 的版位會直接顯示「取件限定優惠」。例如：「大杯拿鐵第2杯半價」的條碼直接顯示在認證條碼下方。  
  6. **互動邏輯：** 店員掃描上方的認證條碼完成取件核對，順手即可掃描下方的優惠條碼進行結帳。用戶無需切換頁面，無需尋找優惠券，整個流程一氣呵成。

#### **2.2.3 取件後歸戶 (Auto-Deposit) 機制**

傳統的小白單（收據優惠券）存在遺失率高、兌換率低的問題。OPENPOINT 引入了數位歸戶機制 2。

* **流程：** 店員刷讀取件條碼後，系統後端會自動觸發「購物抵用券」的發放。  
* **UI 回饋：** 用戶手機會再次收到推播：「您有一張取件優惠券已歸戶至您的票券匣」。  
* **價值分析：** 這種「自動歸戶」機制創造了「為了用掉優惠券而再次進店」的循環（Retention Loop）。用戶不需要當下決定是否購買，優惠券存在 App 裡會持續產生召回（Recall）效果。

### **2.3 ibon 繳費服務的情境閉環**

針對繳費服務，OPENPOINT 設計了「App 預約、門市結帳、事後回饋」的閉環，試圖將 ibon 的線下流量導回 App 3。

#### **2.3.1 觸發點與時間差攻擊**

OPENPOINT 利用繳費後的「心理放鬆時刻」進行行銷。

* **機制：** 用戶若直接去 ibon 機台操作，App 可能無法即時抓取數據。因此，7-ELEVEN 鼓勵「App 內操作」。  
* **UI 互動流程：**  
  1. **觸發點 (Trigger)：** 用戶在 App 內設定「繳費通知」（如停車費、水電費），獎勵 1 點 OPEN POINT。  
  2. **行動 (Action)：** App 推播「帳單即將到期」。用戶點擊推播，在 App 內產出繳費條碼（或導引至 ibon 機台登入）。  
  3. **獎勵 (Reward)：** 收到推播後，於「隔日 23:59 前」完成繳費，加贈 3 點。  
  4. **視覺回饋 (Visual Feedback)：** 完成繳費後，App 會跳出全螢幕賀卡：「繳費成功！恭喜獲得 3 點」。  
  5. **導流 (Conversion)：** 賀卡下方按鈕直接導向「行動隨時取」的咖啡購買頁面，文案通常為「繳完費輕鬆一下，咖啡寄杯 5 折起」。

#### **2.3.2 ibon 機台與 App 的導流機制 (Landing Mechanism)**

OPENPOINT 極力避免用戶「只用 ibon，不用 App」。

* **列印服務整合：** 用戶在 App 上傳文件 \-\> 獲得 QR Code \-\> 去 ibon 掃描列印。  
* **閉環設計：** ibon 機台操作畫面結束時，螢幕會顯示一個動態 QR Code，文案為：「掃描領取本次交易點數/發票」。用戶必須打開 App 掃描，這強迫了 App 的開啟率 (Open Rate)，並將一次性的機台操作轉化為可追蹤的會員數據。

**3\. 萊爾富 (Hi-Life) 與 OK Mart：差異化導流與生存策略**

相較於 7-ELEVEN 的龐大生態系，萊爾富與 OK Mart 無法在數據量級上正面對抗，因此採取了更具針對性的「利基市場」與「票券經濟」策略。

### **3.1 萊爾富 (Hi-Life)：異業合作與高黏著度票券**

萊爾富的 App 策略側重於\*\*「票券經濟 (Ticket Economy)」**與**「異業導流」\*\*，將 App 打造為一個跨通路的優惠聚合器。

#### **3.1.1 Hi-Coupon 與異業合作的高開啟率策略**

萊爾富深知店數不如雙雄，因此積極與非超商業者結盟 4。

* **合作案例：** 漢堡王、肯德基、拿坡里等餐飲券，以及特色服務（如自助洗衣複合店）。  
* **UI 互動流程 (整買零取/異業券)：**  
  1. 用戶進入「整買零取」或「票券匣」。  
  2. **高開啟率維持：** 這些餐飲券通常折扣極深（如漢堡王買一送一），吸引用戶即使不去萊爾富，也會保留 App 以便去速食店時使用。  
  3. **導流回店 (Cross-Selling UI)：** 在票券的「使用規則」或「核銷頁面」下方，萊爾富會置入「搭配飲料優惠」。例如：使用漢堡王券時，App 介面下方顯示「萊爾富可樂加購價 19 元，憑券核銷」。  
  4. **數據洞察：** 這讓萊爾富掌握了用戶的「外食偏好」，即使該消費不發生在店內，也能豐富用戶標籤（如：喜愛速食的年輕族群），進而推播相關的零食或飲料優惠。

#### **3.1.2 包裹查詢與二次銷售的 UI 設計**

萊爾富的 App 在包裹查詢流程中，展現了與 7-ELEVEN 不同的設計思維。

* **UI 描述：**  
  * 點擊「Life-ET 服務」或「包裹查詢」。  
  * **版位設計：** 包裹狀態顯示在上方（約佔 30% 畫面），下方 70% 則被設計為「取件加購區」。  
  * **互動細節：** 當狀態顯示為「待取件」時，該區塊會變成紅色或亮色強調，並附帶按鈕「查看取件專屬優惠」。這與 7-ELEVEN 類似，但萊爾富更強調「高單價異業券」的交叉銷售（例如：取件送洗車折價券或特定手搖飲優惠），試圖提高單次進店的客單價 (Basket Size)。

### **3.2 OK Mart：IoT 機台與 App 的弱連結/強功能**

OK Mart 的店數最少，其策略是依賴 **OK go (IoT Kiosk)** 與 **OK Mart App** 的功能性連結，特別是針對「跨平台網購族群」。

#### **3.2.1 OK go 導流與取件行銷**

由於 OK Mart 經常與蝦皮等電商平台合作推出免運活動，其 App 高度聚焦於「物流狀態」。

* **針對「取件/繳費」族群的行銷：**  
  * OK Mart App 的首頁非常「功能導向」，強調「OK go」功能模組。  
  * **導流 UI：** 當用戶在 App 內完成「預選繳費單」或「查詢取件」後，生成的條碼頁面下方，會常駐「小白單專屬優惠」預覽。  
  * **弱點分析：** OK Mart 依然較依賴實體小白單（收據行銷），App 更多時候扮演的是「條碼載體」角色，缺乏像 7-ELEVEN 那樣即時的「點數回饋閉環」或「自動歸戶」機制，這導致用戶在取件完畢後極易關閉 App，留存率較低。

**4\. 第三方發票 App (發票怪獸/AIFIAN)：攻擊型截流 (Conquesting) 策略**

這是本報告對 My FamiPort 最具戰略價值的章節。第三方發票 App 擁有零售商夢寐以求的\*\*「跨通路數據 (Cross-Channel Data)」\*\*。他們知道用戶在 7-ELEVEN 買了什麼，這是 FamilyMart 自身數據無法觸及的盲區。

### **4.1 攻擊型截流 (Conquesting) 的運作機制**

「截流」的核心在於：**識別競品用戶 \-\> 驗證消費能力 \-\> 推送轉換誘因**。

#### **4.1.1 識別與驗證 (Identification & Verification)**

* **數據源：** 財政部電子發票 API 5。當用戶掃描發票或綁定載具時，App 會解析出 Seller Name (如：統一超商) 與 Details (如：CITY CAFE)。  
* **關鍵字觸發 (Keyword Targeting)：** 發票怪獸無法直接在 App 介面上大喇喇地寫「掃 7-11 發票送全家券」（可能涉及商標爭議），但他們透過\*\*「任務關鍵字」\*\*來繞過。後台邏輯設定為：當發票明細包含 "CITY CAFE" 或 "統一" 字串時，觸發特定獎勵邏輯。

#### **4.1.2 具體 UI 互動描述：遊戲化包裝 (Gamification)**

發票怪獸 (Invoice Monster) 是此領域的佼佼者，他們利用「怪獸養成」降低用戶對隱私的戒心 6。

* **案例場景：** 鎖定 7-ELEVEN 咖啡重度使用者，導流至 FamilyMart。  
* **UI 流程 (The "Monster" Loop)：**  
  1. **掃描/匯入：** 用戶掃描一張 7-ELEVEN 的發票。  
  2. **觸發回饋：** 畫面上的一隻萌寵（怪獸）做出「吃掉發票」的動畫，並吐出「怪獸幣」。  
  3. **隱性任務觸發 (Backend Logic)：** 系統偵測到這是一張「競品發票」。  
  4. **彈窗/對話 (Engagement)：** 怪獸跳出對話框：「主人，這家咖啡好像喝膩了？這隻新怪獸想嚐嚐別的味道！」（文案隱晦但具引導性）。  
  5. **獎勵引導 (Conquesting Offer)：** 畫面顯示：「解鎖限時任務：至『全家便利商店』消費任一咖啡品項，掃描發票，怪獸幣 **10倍送**（或直接送 10 元全家購物金）。」  
  6. **閉環驗證：** 用戶真的去全家買了咖啡並掃描發票 \-\> 系統驗證成功 \-\> 發放高額獎勵。

### **4.2 數據變現與精準受眾包 (Audience Segmentation)**

第三方 App 會將這些數據打包成「受眾包 (Audience Segment)」賣給廣告主（例如 My FamiPort 團隊）。

* **Segment A (Loyalist)：** 「7-11 忠誠客」 (月消費 \> 10次，佔比 \> 80%)。 \-\> **策略：** 強力首購優惠（破冰）。  
* **Segment B (Switcher)：** 「雙棲客」 (7-11 與 全家 各半)。 \-\> **策略：** 差異化商品推播（如 Fami\!ce 霜淇淋）來增加佔比。  
* 實際操作搜尋 6： 發票怪獸有「商城」機制，用戶用怪獸幣兌換「全家購物金」。這本身就是一種導流。雖然公開搜尋未見赤裸裸的「掃 7-11 送全家」官方新聞，但其商業模式允許廣告主投放「指定通路任務」。

**5\. 工具流 UX 橫向對比 (包裹與服務)**

當用戶處於「功能性操作」模式時，對廣告的容忍度最低，但也最專注。如何在這短短幾秒內置入行銷資訊是 UX 設計的關鍵。

### **5.1 包裹查詢流程：In-App vs. Web Redirect**

| 特性 | 7-ELEVEN (OPENPOINT) | FamilyMart (My FamiPort) | 萊爾富 / OK Mart |
| :---- | :---- | :---- | :---- |
| **技術架構** | **Native In-App (原生)** | **Native In-App (原生)** | **Hybrid / Web View (網頁跳轉)** |
| **流暢度** | 高 (無白屏加載) | 高 (整合 150+ 平台 8) | 中/低 (需等待網頁載入) |
| **廣告版位** | **原生嵌入 (Native Ads)** | 頂部/底部 Banner | 彈窗 (Pop-up) 或 蓋板 |
| **主要優勢** | 0元認證條碼直接整合優惠券 | 跨平台一站式追蹤 (蝦皮/Momo) | 結構簡單 |
| **體驗斷點** | 無 (閉環完整) | 無 | 有 (跳轉時易流失) |

#### **5.1.1 競品 UI 比較與機會點**

* **7-ELEVEN：** 在查詢結果頁，每一筆包裹資訊下方都預留了\*\*「Accordion (手風琴展開)」\*\*區塊。用戶點擊詳情時，除了物流進度，還會展開「相關推薦商品」。這種「原生嵌入」比蓋板廣告更不惱人，且讀取速度極快。  
* **萊爾富/OK：** 傾向於使用 **Web Redirect**。當用戶點擊查詢時，會開啟一個內嵌瀏覽器視窗。這導致無法利用 App 的推播權限做即時互動，且廣告形式多為「置底固定 Banner (Sticky Bottom Banner)」，容易被誤觸或忽視，甚至因為加載速度慢而被用戶直接關閉。

### **5.2 繳費服務流程與 UI 安插**

* **痛點：** 輸入長串帳單號碼。  
* **競品解法：** 7-ELEVEN 與 FamilyMart 皆支援「智慧掃描」 (OCR) 或「條碼掃描」。  
* **行銷置入 UI：**  
  * **最佳實踐 (Best Practice)：** 在「掃描成功」後的確認頁面 (Confirmation Page)。這是用戶情緒最放鬆的一刻。  
  * **UI 建議：** 不要在此頁放無關廣告。應放「相關聯」廣告。例如繳完「信用卡費」，推播「指定銀行信用卡綁定 FamiPay 回饋」。7-ELEVEN 在此處的做法是直接顯示「點數回饋進度條」，激勵用戶完成最後一步支付。

## ---

**6\. 給 My FamiPort 的優化建議與結論**

基於上述地毯式調查，針對 My FamiPort 的「通知優化」與「截流」目標，提出以下策略建議：

### **6.1 解決「通知騷擾」：從廣播走向觸發 (From Broadcast to Trigger)**

借鏡 7-ELEVEN 的成功經驗，應停止全網發送通用的促銷訊息。

* **策略：** 建立「行為觸發池 (Event Trigger Pool)」。  
* **具體執行：**  
  * **Trigger：** 用戶剛「取件成功」。  
  * **Action：** 1分鐘內推播：「包裹重嗎？辛苦了！來全家喝杯茶休息一下（附 5 元折價券）」。  
  * **Benefit：** 這種通知被視為「關懷」而非「廣告」。

### **6.2 利用發票數據進行截流 (Data-Driven Conquesting)**

不能僅依賴自家 App 數據，必須與第三方 App (如發票怪獸) 進行深度戰略合作 (Data Partnership)。

* **戰術一：競品客群清洗**  
  * 與發票怪獸合作，圈選「過去 3 個月曾在 7-ELEVEN 消費咖啡 \> 5 次」的用戶 ID。  
  * 在發票怪獸 App 內投放「全家 Let's Café 試飲券」任務，但限制必須使用 FamiPay 支付才能解鎖，以此強迫競品用戶下載並使用 My FamiPort。  
* **戰術二：購物籃分析截流**  
  * 分析競品發票中，「常與御飯糰一起購買的飲料」是什麼？  
  * 若發現是某牌綠茶，則在 My FamiPort 推播中強調「全家御飯糰 \+ 該牌綠茶」的組合優惠，文案強調「我們也有，而且更便宜/更好吃」。

### **6.3 總結**

台灣超商 App 戰場已從「功能戰」轉向「數據戰」。7-ELEVEN 透過 OPENPOINT 築起了強大的生態護城河，將每一個物理動作都數位化。My FamiPort 若要突圍，除了優化自身的包裹/繳費 UX (如一站式追蹤) 外，更應積極利用外部數據 (Third-party Data) 進行攻擊型行銷，將對手的每一次交易，都轉化為自身的獲客機會。
